# Section Structure — How to fill each one

Which sections to include and their order is defined by CLAUDE.MD (Drive). Here, for each
known section name, is what content goes in and how to format it. If CLAUDE.MD names a
section not listed here, compose it sensibly following the same style. The section
headings below are Spanish because they are the report's actual headings.

## Encabezado
- Title: "Informe de Evolución Kinesiológica".
- Report date (distinct from the session date if applicable).
- A horizontal separator line below.

## Datos del Paciente
Two-column table (Campo | Detalle):
- Full name
- Age (years)
- DNI (with dots: XX.XXX.XXX)
- Kinesiological diagnosis (functional, NOT medical)

## Datos de la Sesión
Two-column table:
- Session date
- Duration (minutes)
- Type: Evaluación inicial / Tratamiento / Control / Re-evaluación
- Session number, if known: "N de M"

## Tratamiento Realizado
Sub-headings (###) per category, only those that apply:
- **Ejercicios terapéuticos:** each exercise with series × repetitions × load; range if relevant.
- **Elongación:** muscle, technique (estática/dinámica/FNP), hold time.
- **Técnicas manuales:** type, area, duration.
- **Agentes físicos:** modality, area, duration, parameters.

## Evaluación del Progreso
Use comparison tables if there's prior data (see `comparacion-progreso.md`):
- **ROM:** movement | previous | current | difference
- **Dolor (EVA 0-10):** situation | previous | current
- **Fuerza muscular:** Daniels scale (0-5), indicating side
- **Observaciones clínicas:** passive-voice text ("Se observa…", "Se evidencia…").

## Plan de Tratamiento
- **Objetivos a corto plazo:** measurable, with a deadline.
- **Objetivos a largo plazo:** functional.
- **Indicaciones:** session frequency, home program, restrictions.

## Observaciones
- Free text: referrals, contraindications, relevant events.
- Close with space for signature and license number (a placeholder for the professional to fill in).

## General format
- Tables for structured data and comparisons.
- Bullets for exercises and treatments.
- Bold for labels.
- `###` for sub-sections inside Tratamiento and Progreso.
- `---` between main sections.
- If a section has no data: omit it or write "Sin datos", per the user's preference.
